1. Overview & Purpose

This Privacy Policy describes how Joules to Watts ("J2W", "we", "us", or "our") collects, uses, and protects information when you use the Source Assist Chrome Extension ("the Extension", "Source Assist").

Source Assist is an AI-powered recruitment tool that helps J2W recruiters extract structured consultant profile information from text, images, and PDF résumés. The extracted data is formatted for direct pasting into J2W's internal Excel-based tracking system.

By installing or using the Extension, you agree to the collection and use of information in accordance with this policy.

2. Scope & Applicability

Internal Enterprise Tool Only. Source Assist is an internal tool exclusively for employees of Joules to Watts. It is not available to or intended for the general public. Access is technically enforced — only email addresses ending in @joulestowatts.com or @joulestowatts.co can register or log in.

This policy applies solely to the Source Assist Chrome Extension published on the Chrome Web Store. It governs all data processing that occurs through the Extension and its supporting backend infrastructure.

3. Single Purpose Statement

Source Assist has a single, clearly defined purpose:
To extract structured consultant profile data (name, contact, skills, experience, etc.) from recruiter-submitted text, image screenshots, and PDF résumés using AI, and copy the result to the clipboard in a format ready for pasting into Excel.

The Extension does not perform any other function. It does not monitor browsing activity, inject content into pages, capture keystrokes, track user behaviour outside the Extension panel, or serve advertisements.

10. Remote Code Policy

Source Assist does not use remote code.

All JavaScript executed by the Extension is contained within the extension package submitted to the Chrome Web Store. No code is downloaded, injected, or executed from external URLs at runtime. The Extension communicates with its own backend API to receive data (extraction results), not to receive or execute code. This complies fully with the Chrome Web Store's Remote Code policy.

12. Security Measures

  • Encrypted transport — All communication between the Extension and the backend API is over HTTPS/TLS.
  • Password hashing — Passwords are stored exclusively as bcrypt hashes with a unique per-user salt. Plaintext passwords are never stored, logged, or transmitted.
  • Signed sessions — Authentication uses signed HS256 JWT tokens with a 30-day expiry, stored only in chrome.storage.local on the user's device.
  • Email verification — Account registration and password reset both require a 6-digit OTP sent to the user's work email, which expires in 10 minutes and is invalidated immediately after use.
  • Domain restriction — The backend rejects all requests from email addresses outside @joulestowatts.com and @joulestowatts.co at the server level, enforced independently of the client.
  • Database security — User data is stored in a managed PostgreSQL database on Supabase, which provides encryption at rest, access controls, and regular backups.
